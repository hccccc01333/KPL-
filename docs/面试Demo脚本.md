# 面试 / Demo 讲解脚本（KPL 实时胜率）

> 目标：5–8 分钟讲清「业务价值 → 技术难点 → 诚实评估 → 坏局修复故事」  
> 打开 Dashboard：`streamlit run scripts/dashboard.py`  
> 当前模型：**V11**（Early RF + Midlate Voting + 金差一致性护栏）

---

## 0. 开场一句话（15 秒）

不是做一个「赛后猜胜负」的模型，而是面向解说/导播的**实时胜率中控**：谁赢面大、为什么、转折点在哪。

---

## 1. 产品链路（30 秒）

```text
官方赛程 API → 自动识别 battle → 局内快照采集
→ 统一 FeatureBuilder（训练=线上）
→ V11 分阶段预测 + 校准 + 金差护栏
→ Dashboard：实时中控 / 历史复盘 / 战报拐点
```

强调：**特征与预测入口只有一套**（`kpl_official_core`），避免 notebook 与线上漂移。

---

## 2. 主指标怎么报（必背）

**不要主推混合 AUC。** 面试口径：

| 阶段 | Holdout Brier（约） | 含义 |
|------|---------------------|------|
| early ≤8′ | **~0.057** | 早期不确定，看概率是否靠谱 |
| mid 9–15′ | **~0.036** | 中期最稳 |
| all | **~0.049** | 整局平均 |
| 坏局 Acc | **≥0.86** | 曾翻车的局也要方向正确 |

数据：clean 真实局 holdout（约 8 局），脚本：`python scripts/backtest_realtime_v9.py --gates`

---

## 3. Demo 点击路径（2–3 分钟）

### A. 实时中控（或历史下钻同一套面板）

1. 选一局有完整快照的 clean battle  
2. 指胜率曲线 + 经济差：说明「概率随优势变化」  
3. 指「领先原因」：地图压力 / 资源控制 / 动量 —— 解释型特征  

### B. 坏局故事（核心亮点）

打开诊断报告或复盘局 `501236240_10`（AG 碾压 DYG）：

- **问题**：第 5 分钟金差已 −1284，模型仍报蓝方 ~0.84  
- **根因**：金差护栏原先只在 ≥8′ 生效，早期「和经济拧巴」漏掉了  
- **修复**：5–7′ 增加**轻度**护栏（仅大幅金差 + 方向冲突时拉动）  
- **结果**：该局 Acc **0.82 → 0.91**，holdout early Brier **0.068 → 0.057**，gates 仍 PASSED  

口播金句：护栏是**安全兜底**，不是主模型；理想状态是模型自己少拧巴，护栏命中下降。

### C. 拐点 / 战报（若页面有）

指 1–2 个转折点：暴君 / 推塔后胜率跳变，对应业务「这波之后谁赢面更大」。

---

## 4. 技术追问速答

| 问题 | 答法 |
|------|------|
| 为什么分 early / midlate？ | 早期噪声大、特征不同；6–9′ 软混合，避免硬切换跳变 |
| 为什么有模拟样本？ | 真实局仍少，终局模拟降权补结构；评估只看真实 holdout |
| 为什么不用深度学习？ | 样本百局级、要可解释、可校准；树模型 + 护栏更稳 |
| 因果先验是什么？ | 战队胜率只用**该局开赛前**历史，避免泄漏 |
| clean / quarantine？ | 质量分层；训练默认 clean，raw 只作入库池 |

---

## 5. 诚实短板（加分）

- clean 真实局仍约 **百局**，继续采集比再调参更重要  
- mid/late **ECE** 仍偏高，绝对值播报需低置信提示  
- 开局 1′ 金差几乎为 0 时方向可能错（不可用未来信息硬修）

---

## 6. 收尾（15 秒）

「端到端闭环 + 诚实分阶段评估 + 坏局可诊断可修复」——这是数据分析/算法实习最想看到的工程闭环，而不只是一个 AUC 数字。
