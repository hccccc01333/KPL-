# V1 阶段必学技术

## 1. requests + JSON 解析

### 思路

爬虫的本质就两件事：**发请求** + **解 JSON**。掌握三个核心点就能开工：
- `requests.get()` 怎么带 headers / params / cookies
- `response.json()` 拿到的就是 dict / list 嵌套结构
- 异常处理：超时、状态码非 200、JSON 解析失败

### 验收标准

能独立完成下面这件事就算学够了：

> "给我一个 URL，我能在 5 分钟内调通接口、把返回 JSON 存成本地文件、提取我要的字段，并且能处理超时和报错。"

### 推荐资源

- 中文入门：[菜鸟教程 · Python requests](https://www.runoob.com/python3/python-requests.html)（30 分钟过完）
- B 站：搜"requests 库 python 教程"，挑播放量最高的 1-2 小时课程
- 官方：[Requests 中文文档](https://requests.readthedocs.io/projects/cn/zh-cn/latest/)（有问题再查）

### 练手小任务

```python
import requests, json
url = "https://httpbin.org/json"
r = requests.get(url, timeout=5)
print(r.status_code, r.json())
```

跑通这 4 行，再把它改成"超时重试 3 次 + 失败返回 None"——你就够开工了。

### 踩坑提示

- ⚠️ 一定要加 `timeout=`，否则程序可能卡死
- ⚠️ `response.json()` 解析失败会抛异常，要 try/except
- ⚠️ 中文打印乱码：`json.dumps(data, ensure_ascii=False)`

## 3. pandas 进阶（json_normalize + 时序滚动）

### 思路

项目1 你练过 pandas 基础（groupby、value_counts、pivot_table）。项目2 多两个新技能：

**新技能 A：嵌套 JSON → DataFrame**
```python
pd.json_normalize(data, record_path=['battle_player_list'], meta=['battle_id'])
```

**新技能 B：时间窗口滚动特征**——这是数据泄漏防护的核心。
```python
# 按队伍分组 + 按时间排序 + 用前 N 场算均值
df.groupby('team').apply(
    lambda g: g.sort_values('date').rolling(5).mean()
)
```

### 验收标准

> "我能把一个嵌套 3 层的 JSON 转成 DataFrame，并且能用 rolling/expanding 给每场比赛算出'该队前 5 场的场均经济'，并且严格不用未来数据。"

### 推荐资源

- 项目1 的 `pandas库学习/` 三个 notebook 是基础
- 知乎：搜"pandas 时间序列 rolling"
- B 站：搜"pandas 高级教程"，挑 3-5 小时长课
- 官方：[pandas 用户指南 · 时间序列](https://pandas.pydata.org/docs/user_guide/timeseries.html)

### 练手小任务

造一份 50 行假数据（10 支队 × 5 场比赛），用 `groupby + rolling(3).mean()` 算每场比赛对应的"该队前 3 场场均得分"，**并用断言验证没有未来数据泄漏**（第 1 场该是 NaN，第 2 场只用前 1 场）。

### 踩坑提示

- ⚠️ rolling 默认会带上**当前行**——要先 `shift(1)` 错开
- ⚠️ 时间排序要在 groupby 内部做，否则会乱
- ⚠️ 大数据量用 `transform` 比 `apply` 快 10 倍

## 5. logging 模块

### 思路

- 分级别打日志（DEBUG / INFO / WARNING / ERROR）
- 同时输出到控制台 + 文件
- 关掉 DEBUG 不影响程序

最少要会下面这个模板：

```python
import logging
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s [%(levelname)s] %(message)s',
    handlers=[
        logging.FileHandler('logs/crawler.log', encoding='utf-8'),
        logging.StreamHandler()
    ]
)
logger = logging.getLogger(__name__)
logger.info("爬虫启动")
```

## 7. scikit-learn 标准流程

### 思路

老项目用了 sklearn 但有 3 个 bug。学这一节就是为了**写出"教科书级"的标准流程**：

```python
# 1. 拆分（一定要 stratify）
X_tr, X_te, y_tr, y_te = train_test_split(X, y, test_size=0.2, stratify=y, random_state=42)
# 2. 标准化（只在训练集 fit）
scaler = StandardScaler().fit(X_tr)
X_tr_s = scaler.transform(X_tr); X_te_s = scaler.transform(X_te)
# 3. 训练
model = LogisticRegression().fit(X_tr_s, y_tr)
# 4. 评估
from sklearn.metrics import classification_report
print(classification_report(y_te, model.predict(X_te_s)))
# 5. 交叉验证
from sklearn.model_selection import cross_val_score
cv = cross_val_score(model, X_tr_s, y_tr, cv=5, scoring='roc_auc')
print(f'AUC: {cv.mean():.3f} ± {cv.std():.3f}')
```

5 个模型（LR / SVM / RF / KNN / XGBoost）都套这个模板。

### 验收标准

> "我能默写出标准 ML pipeline，并且能解释为什么标准化只在训练集 fit、为什么要用 stratify。"

### 推荐资源

- B 站：搜"sklearn 教程"，黑马 / 莫烦 / 跟李沐学AI 都行
- 中文文档：[sklearn 中文教程](https://www.scikitlearn.com.cn/)
- 必看：[Hands-On Machine Learning（中译版）](https://github.com/ageron/handson-ml3)第 1-3 章

### 踩坑提示

## 9. 时序特征工程 ⭐ 最重要

### 思路

> 这是项目2 修复老项目最大 bug 的核心技能，**也是面试官最爱问的反向题**。

核心概念：
- **数据泄漏（Data Leakage）**：用未来信息预测过去 = 假高分
- **时间窗口特征**：用 `t-N 到 t-1` 的数据预测 `t` 时刻
- **冷启动问题**：前 N 场没有历史，怎么处理？

### 推荐资源

- **必读**：[Kaggle 教程 · Time Series](https://www.kaggle.com/learn/time-series)（英文但短，2 小时过完）
- 知乎搜"数据泄漏 特征工程"
- 知乎专栏《风控建模》系列文章里的"时间窗口特征"
- 中文：B 站搜"特征工程 时间窗口"

### 练手小任务

### 踩坑提示

## 11. 蒙特卡洛模拟

### 思路

BO7 比分预测的精髓：单局胜率 65% **不等于** BO7 4-0 胜率 65%。要用模拟：
- 每局按胜率 65% 抽签 → 直到一方拿到 4 胜
- 重复 10000 次 → 统计 4-0 / 4-1 / 4-2 / 4-3 / 0-4 等比分的频率

```python
import numpy as np
def simulate_bo7(p_a_win, n_sim=10000):
    scores = []
    for _ in range(n_sim):
        a, b = 0, 0
        while a < 4 and b < 4:
            if np.random.rand() < p_a_win: a += 1
            else: b += 1
        scores.append(f"{a}-{b}")
    return pd.Series(scores).value_counts(normalize=True)
```

### 验收标准

> "我能在 10 行内写出 BO7 比分概率模拟，并能加'状态波动'扰动让模型更真实。"

## 12. SHAP 可解释性

### 思路

> "这个模型为什么预测 A 队赢？" —— SHAP 给每个特征算一个**贡献度**。

核心 3 个图：
- `summary_plot`：全局特征重要性 + 方向
- `force_plot`：单条样本"为什么这么预测"
- `dependence_plot`：单个特征对预测的影响曲线

### 验收标准

> "我能用 SHAP 给一场具体比赛画图，并用人话解释'模型最看重的 3 个特征是什么、它们如何让胜率从 50% 偏移到 70%'。"

### 推荐资源

- 知乎搜"SHAP 通俗讲解"
- 官方 GitHub：[slundberg/shap](https://github.com/shap/shap) 看 README 的图
- B 站搜"SHAP 实战"

### 踩坑提示

- ⚠️ Tree 模型用 `TreeExplainer`，线性模型用 `LinearExplainer`，黑盒模型用 `KernelExplainer`（慢）
- ⚠️ Python 3.14 装 SHAP 可能装不上 → 用 sklearn 的 `permutation_importance` 替代

# 避坑指南：学习时常见的弯路

| 弯路 | 正确姿势 |
|---|---|
| 想等"完全学会"才开工 | **边做边学**——任务驱动，遇到不会再查 |
| 教程一刷到底 | 看 30 分钟 → 立刻动手敲一遍 → 卡住再回来看 |
| 只看视频不动手 | 视频只看 30%，70% 时间在 IDE 里写代码 |
| 抄完代码不复盘 | 每抄一段问自己："这里为什么这么写、删掉一行会怎样" |
| 一上来就调超参 | 先跑通 baseline，再优化 |
| 看了 5 种 ML 库不知道选哪个 | 项目2 只用 sklearn + xgboost，**别贪多** |
| 想"完美"的特征工程 | 先有 20 个粗糙特征，比纠结 1 个完美特征好 |

# 推荐的 4 个"学习压舱石"资源

如果只能选 4 份资料，就这 4 份：

1. **《Python for Data Analysis》（Wes McKinney 著，pandas 作者本人）** —— pandas 圣经
2. **《Hands-On Machine Learning, 3rd》（Aurélien Géron）** —— sklearn / XGBoost 实战圣经
3. **Kaggle Learn 系列**（[kaggle.com/learn](https://www.kaggle.com/learn)）—— 短小精悍，2 小时一个主题
4. **B 站「跟李沐学AI」频道** —— 中文 ML/DL 课程标杆
